atom grammar + visibility lever.

One PR off master: the discriminated-union atom grammar baseline, the done:stale atom that justifies its first concrete user payoff, and a hairline lever that hides done tasks older than 24h. Designed so future atoms (tag:, has:body, root, under:title) drop in without rework.

Reviewed by Lowy — verdict folded below. Headlines: ship the union with three witness kinds; fix three pre-existing volatility leaks alongside; round-trip is load-bearing; keep query session-only.

02 Atom grammar — shared/atoms.ts

One file. Single volatility axis (atoms change together). Three kinds — enough to make the discriminated union load-bearing, not so many that the grammar bundles unrelated data-model work.

// shared/atoms.ts
export type Atom =
  | { kind: "text";   needle: string }                              // trivial witness
  | { kind: "done";   value: "no" | "yes" | "fresh" | "stale" }      // structured witness
  | { kind: "not";    inner: Atom };                              // recursive witness

export const STALE_THRESHOLD_MS = 24 * 60 * 60 * 1000;

export function parseAtoms(query: string): Atom[];
export function serializeAtoms(atoms: Atom[]): string;       // round-trip with parseAtoms
export function evalAtom(atom: Atom, task: Task, now: number): boolean;
export function evalAtoms(atoms: Atom[], task: Task, now: number): boolean;  // AND

Lowy (A — phase split) → "done:X is the structured-kind witness; not is the recursive witness; text is the trivial witness. Three is enough to make the discriminated union load-bearing." Adding tag: or has:body here would bundle unrelated data-model volatility into one PR.

Hickey (F1, carried forward) → Closed enum on the done axis, four values, one done:X per query. Treating done:stale as a sibling of done:no / done:yes silently admits the impossible combination done:no done:stale (empty set) and the redundant done:yes done:stale. done:stale ⊂ done:yes — refinement, not a peer.

Lowy (C — done:fresh) → Include all four enum values, including fresh. Even though only stale has a caller in this PR: "Closed enums should be exhaustive on their axis. The cost of the symmetric variant is ~6 lines; the cost of adding it later is a parser change, a serializer change, a test refresh, and a discoverability hit."

Hickey (F4) + Lowy (F — round-trip) → serializeAtoms ∘ parseAtoms = id on canonical inputs. The lever's deactivate path is parse → filter out done:stale → serialize. If round-trip isn't identity, toggling the lever rewrites the user's typed query. Property-test it in this PR (fast-check or hand-rolled fuzz) — the cheapest insurance against the most likely regression.

Lowy (D — module boundaries) → Single shared/atoms.ts is right for this PR. Split into per-kind files starts paying for itself only when atom kinds have non-trivial per-kind evaluation logic (e.g. tag: needing a tag-storage join). Neither threshold is hit. STALE_THRESHOLD_MS belongs here, not next to DRAG_LONGPRESS_MS in schemas.ts:49-55 — different volatility axis.

03 Data model — completedAt

Today's updatedAt bumps on rename, move, and toggle (storage/tasks.ts:211-220, 232-237, 253-257) — can't proxy "completed at". One new column, written only inside the existing toggle transaction.

SiteChange
migrations/<new>.ts ADD COLUMN completed_at TEXT + CHECK (status = 'done' OR completed_at IS NULL). Legacy rows backfill NULL — predicate treats NULL as "not stale", so existing done tasks remain visible until the user re-toggles them.
storage/schema.ts Add completed_at: string | null to TasksTable.
shared/schemas.ts completedAt: z.string().nullable() on TaskSchema. BackupSchema inherits via TaskSchema.
storage/tasks.ts — toggle (:243-260) Both halves. todo→done: set completed_at = now. done→todo: set completed_at = null. The clear-on-untoggle makes the stale set a ratchet, not a sliding window.
storage/tasks.ts — upsert (:92-115) Map task.completedAt into .values() and .doUpdateSet() — backup/restore rides this path.
storage/tasks.ts — rowToTask (~:55) Read r.completed_at → completedAt.

Hickey (F3, DB-level guard) → The CHECK constraint enforces the invariant at the SQLite layer — free, one extra line. Without it, a future direct-edit path could orphan the field silently.

Lowy (E — parse-on-read) → "query() is the single source of truth. A parallel leverActive signal kept in sync via effect creates a second receptacle for the same volatility axis — any divergence is a correctness bug. If profiling ever shows the parse cost matters (it won't), wrap in createMemo. Do not introduce a separate signal."

Lowy (G — persistence) → Session-only signal. No URL hash, no localStorage. "Both are premature commitments to a query-string serialization format that will change as new atom kinds land. Each new atom is a new round-trip surface; locking the format into a URL contract means breaking deep links every time the grammar grows." Revisit once the grammar stabilizes.

Philosophy → Relational, not calendrical. The atom says "the moment has passed", not "you are late". 24h is a grace window, not a deadline. Compatible with the README's eternalist stance — the rule it bans is "Today/This week/Overdue smart lists"; this is the opposite axis.

05 Pre-existing leaks fixed in the precursor PR
#LeakFix
1 client/highlight.ts rebuilds a RegExp to find match boundaries; shared/filter.ts uses .includes() for the boolean. Two implementations of "where does the needle hit". When evalAtoms lands, highlightSegments will silently fall out of sync. Export matchPositions(title, needle): {start, end}[] from shared/filter.ts. Both matchesQuery (boolean reduction) and highlightSegments (segment rendering) call it. Single computation site.
2 byParentMap, sortedWithDepths, siblingsOf, resolveKeyMove are free functions at the top of client/App.tsx. Meanwhile shared/tree.ts already holds ancestorIds / descendantIds. Same conceptual operation (parent-pointer graph walks) split across two files on no principled axis. Move them into shared/tree.ts. App.tsx becomes a pure orchestrator.
3 client/App.tsx:208-227 hardcodes the "show matches + their ancestors, dim the ancestors" policy inline in the rows memo. Not testable, not re-callable from evalAtoms. Extract applyFilter(sorted, predicate, now): Row[] as a pure function. The rows memo becomes a one-liner. Predicate is named, testable, re-callable — making the feature PR's evalAtoms integration trivial.
06 Forward extensibility

When future atoms land → Each new atom kind is one new variant in the union + one new branch in parseAtoms + one new branch in evalAtom. The data-model dependencies travel with the atom that needs them — tag: ships with tag storage; has:body ships with body content; root / under:title are pure tree walks via the already-consolidated shared/tree.ts. The grammar PR closes; the data-model PRs that bring each new atom open separately.
